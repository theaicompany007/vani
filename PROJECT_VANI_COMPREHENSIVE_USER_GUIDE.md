## Introduction

### What is Project VANI?

Project VANI (Virtual Agent Network Interface) is a Flask-based outreach command center that enables multi-channel communication (Email, WhatsApp, LinkedIn) with AI-powered message generation, pitch presentations, and target management. The platform supports multi-industry tenant isolation and provides comprehensive tools for sales outreach, prospecting, and relationship management.

### Key Capabilities

- **Multi-Channel Outreach**: Email, WhatsApp, LinkedIn messaging
- **AI-Powered Content**: Automated message and pitch generation
- **Industry Management**: Multi-industry support with tenant isolation
- **Target Management**: Comprehensive CRM for companies and contacts
- **Knowledge Base**: Self-learning system with RAG integration
- **Analytics & Tracking**: Real-time engagement metrics
- **Meeting Scheduling**: Cal.com integration
- **Bulk Operations**: Import/export via Excel and Google Sheets

## Super Administrator Guide

### Role Overview

Super Administrators have complete control over the VANI platform infrastructure, including:
- System deployment and configuration
- Database management and migrations
- User and permission management
- Industry setup and configuration
- Integration management (APIs, webhooks)
- System monitoring and maintenance

### Access & Authentication

**How to Become a Super Administrator:**

1. **Via Database (Direct SQL):**
   ```sql
   UPDATE app_users 
   SET is_super_user = true 
   WHERE email = 'your-email@example.com';
   ```

2. **Via Setup Script:**
   ```bash
   python do_setup.py
   # Enter your email when prompted
   ```

3. **Via Another Super User:**
   - Have an existing super user log in
   - Navigate to User Management tab
   - Click "Make Super User" for your account

- Manage user permissions within their scope
- Monitor industry-specific activities
- Configure industry-specific settings

## Troubleshooting Guide

### Common Issues & Solutions

**Issue: Cannot log in**
- **Solution:** Verify user exists in `app_users` table
- **Check:** Email and password are correct
- **Fix:** Reset password or create user via super user

**Issue: No targets visible**
- **Solution:** Check industry assignment
- **SQL:** `SELECT * FROM user_industries WHERE user_id = 'user-uuid';`
- **Fix:** Assign industry via User Management

**Issue: Cannot send messages**
- **Solution:** Check permissions
- **Command:** `python scripts/check_user_permissions.py user@example.com`
- **Fix:** Grant `outreach` permission

**Issue: Webhooks not working**
- **Solution:** Verify ngrok is running
- **Check:** `http://localhost:4040/api/tunnels`
- **Fix:** Start ngrok or configure `WEBHOOK_BASE_URL`

**Issue: AI message generation fails**
- **Solution:** Verify `OPENAI_API_KEY` is set
- **Check:** API quota and limits
- **Fix:** Update API key or check billing

## Best Practices

### For Super Administrators

1. **Regular Backups:** Schedule database backups
2. **Monitor Logs:** Check `logs/application.log` regularly
3. **Update Dependencies:** Keep requirements.txt updated
4. **Test Before Deploy:** Always test in staging first
5. **Document Changes:** Keep migration notes

### For Administrators

1. **Industry Setup:** Complete industry setup before assigning users
2. **Permission Audit:** Regularly review user permissions
3. **Knowledge Base:** Keep industry knowledge updated
4. **User Training:** Provide onboarding for new users

### For Users

1. **Target Quality:** Ensure target information is complete
2. **Message Personalization:** Review AI messages before sending
3. **Follow-up:** Track and follow up on engagements
4. **Analytics Review:** Regularly check analytics for insights

### For Business Users

1. **Knowledge Contribution:** Regularly update knowledge base
2. **Feedback:** Provide structured feedback for improvements
3. **Documentation:** Document business requirements clearly
4. **Testing:** Test new features before production use
